# Quote creation fields and product ID

## Product ID (when creating a product)

When you **create a product** in Product Configuration, the app sends:

- **name** – Product name (e.g. "Life Safeguard")
- **code** – Generated as `PROD-` + current timestamp (e.g. `PROD-1739123456789`) so it is unique
- **type** – Category: LIFE, HEALTH, AUTO, HOME
- **isActive** – true

The **product ID** (UUID) is **generated by the backend** when the product is created. The backend returns the saved product including its `id`. The app uses that `id` when creating quotes (as `productVersionId`). The **code** is a human‑readable unique key; the **id** is the internal UUID.

## Quote creation – what each field means

| Field | Meaning |
|-------|--------|
| **Select Product** | The insurance product being quoted (e.g. Life, Auto). Only active products appear. Display is "Product Name (PROD-xxx)". |
| **Applicant first name / last name** | The person who will be the policyholder or main contact for the quote. |
| **Applicant email** | Email address for the quote and future policy communications. |
| **Coverage option ID** | The ID of a **coverage** under this product (e.g. "Critical Illness", "Accidental Death"). Coverages are defined in **Coverage Setup** and are linked to a product/version. Until you have real coverages, a placeholder UUID is used. |
| **Sum insured** | The coverage amount (in currency) for the quote. |

## Coverage vs product

Coverages are **connected to products**: each product has versions, and each product version has **coverage options**. So:

1. Create a **product** in Product Configuration.
2. Define **coverages** in Coverage Setup (and link them to a product/version when the API supports it).
3. When creating a quote, you choose a **product** and a **coverage option** (and sum insured) for that product.
